Convolve_1D (signal, kernel[, opts])
One-dimensional linear convolution using fast Fourier transforms. For certain signal and kernel combinations, this may be significantly faster than signal_ops.linear_convolution.Convolve_1D.

Parameters:

  • signal array Real discrete signal...
  • kernel array ...and kernel.
  • opts table

    Convolve options. Fields:

    • into: If provided, this table will receive the convolution.
    • method: If this is "goertzel", the transforms are done using the Goertzel algorithm, which may offer better performance in some cases. If it is "precomputed_kernel", the kernel argument is assumed to already be FFT'd. If it is "separate", two FFT's are computed separately. Otherwise, the two real FFT's are computed as one complex FFT.

Returns:

    array Convolution.

See also:

    PrecomputeKernel_1D
Convolve_2D (signal, kernel, scols, kcols[, opts])
Two-dimensional linear convolution using fast Fourier transforms. For certain signal and kernel combinations, this may be significantly faster than signal_ops.linear_convolution.Convolve_2D.

Parameters:

  • signal array Real discrete signal...
  • kernel array ...and kernel.
  • scols uint Number of columns in signal...
  • kcols uint ... and in kernel.
  • opts table

    Convolve options. Fields:

    • into: If provided, this table will receive the convolution.
    • method: As per Convolve_1D, but with 2D variants.

Returns:

  1. array Convolution.
  2. uint Number of columns in the convolution. Currently, only the "full" shape is supported, i.e. scols + kcols - 1.

See also:

    PrecomputeKernel_2D

SerialConvolve_1D (sn, kernel, func[, opts])
Enters a loop, fetching a new signal on each iteration.

Each pair of this variable signal and the constant kernel are then convolved, in much the same way they would be by Convolve_1D.

Parameters:

  • sn uint Size of real discrete signal.
  • kernel array Real discrete kernel.
  • func callable Called as

    signal, out = func(sn, arg)

    where signal is a real discrete signal of size sn (additional values beyond this will be ignored) and out will receive the convolution.

    The loop aborts when signal is nil.

  • opts table

    Convolve options. Fields:

    • arg: If provided, this will be the argument to func.
    • method: As per Convolve_1D.
