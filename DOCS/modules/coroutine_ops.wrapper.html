Module coroutine_ops.wrapper

An extended coroutine wrapper behaves like a function returned by coroutine.wrap, though as a loop and not a one-shot call.

Once the body has completed, it will "rewind" and thus be back in its original state, excepting any side effects. It is also possible to query if the body has just rewound.

In addition, a coroutine created with this function can be reset, i.e. the body function is explicitly rewound while active. To accommodate this, reset logic can be attached to clean up any important state.

IsIterationDone (coro)
Queries a coroutine made by Wrap about whether its body just ended an iteration.

Parameters:

  • coro function Wrapper for coroutine to query.

Returns:

    boolean The body finished, and the wrapper has not since been resumed or reset?
Reset (coro, ...)
Resets a coroutine made by Wrap.

If the coroutine is already reset, this is a no-op.

Parameters:

  • coro function Optional wrapper for coroutine to reset; if absent, uses the running coroutine.
  • ... Reset arguments.
Status (coro)
Reports the status of a coroutine made by Wrap.

Parameters:

  • coro function Wrapper for coroutine to query.

Returns:

    string One of the results of coroutine.status, "resetting" during a reset, or "failed_reset" if an error was thrown by Wrap's on_reset.

Wrap (func, on_reset)
Creates an extended coroutine, exposed by a wrapper function.

Parameters:

  • func callable Coroutine body.
  • on_reset callable Function called on reset; if nil, this is a no-op.

    Note that this will be executed in a protected call, within the context of the resetter.

Returns:

    function Wrapper function.
