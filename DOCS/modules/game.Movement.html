Functions

CanGo (index, dir, facing)
Predicate.

Parameters:

  • index uint Tile index.
  • dir string Direction to query.
  • facing string If provided, direction is interpreted as NextDirection(facing, dir).

Returns:

    boolean Can we move to the next tile, going this way?

    Note that this does not say that we can get back from the next tile, i.e. it doesn't guarantee that the tiles are connected, or even that there is a next tile.

    In addition, it does not say whether we can move within this tile, e.g if there is a path from one side to the center.

See also:

    NextDirection
MoveFrom (x, y, dist, dir)
Moves by a given amount, from a given position, in a given direction. This relaxes cornering, and does the heavy lifting of keeping the result "on the rails".

Parameters:

  • x number Current x-coordinate.
  • y number Current y-coordinate.
  • dist number Amount of distance we may move.
  • dir string Direction we want to go.

Returns:

  1. number Result x-coordinate.
  2. number Result y-coordinate.
NextDirection (facing, headed, extra)
Determines next direction, given the direction you're facing and which way you're headed.

Parameters:

  • facing string One of "left", "right", "up", "down".
  • headed string One of "to_left", "to_right", "backward", "forward".
  • extra string

    Optional extra information requested, as additional return value(s):

    • "tile_delta": Index delta to the next tile.
    • "unit_deltas": column- and row-deltas to the next tile.

Returns:

  1. string Absolute direction.
  2. As per extra; otherwise, nil.
  3. As per extra; otherwise, nil.
Turns (swap)

  1. string Normally, "to_left".
  2. string Normally, "to_right".

WayToGo (index, dir1, dir2, dir3, facing)
Chooses which direction to follow at a tile, given some preferences.

Parameters:

  • index int Tile index.
  • dir1 string Preferred direction.
  • dir2 string First runner-up.
  • dir3 string Second runner-up.
  • facing string If provided, the dir* are interpreted as NextDirection(facing, dir*).

Returns:

    string If any of the dir* was open, the most preferred one is returned, without modification; otherwise, returns "backward".
