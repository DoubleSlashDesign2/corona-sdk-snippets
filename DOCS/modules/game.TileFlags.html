Module game.TileFlags

This module deals with tile direction flags.

Terminology-wise, a "flag" in the following refers to an unsigned integer with a single bit set, and a "union of flags" is the logical or of one or more distinct flags.

ResolveFlags (update)
Resolves the current working flags, as set by SetFlags, into the current active flag set. The working flags will remain intact.

The operation begins with the working set of flags, and looks for these cases:

  • One-way, e.g. a right-flagged tile whose right neighbor is not left-flagged.
  • Orphaned border, e.g. a down-flagged tile on the lowest row.

The final result, after culling the troublesome flags, is the resolved flag set.

Parameters:

  • update bool If true, the "flags_updated" event list is dispatched after resolution.

See also:

    game.DispatchList.CallList
Rotate (flags, how)
Reports the state of a set of flags, after a rotation of some multiple of 90 degrees.

Parameters:

  • flags uint Union of flags to rotate.
  • how string If this is "ccw" or "180", the flags are rotated through -90 or 180 degrees, respectively. Otherwise, the rotation is by 90 degrees.

Returns:

    uint Union of rotated flags.
SetFlags (index, flags)
Sets a tile's working flags.

ResolveFlags can later be called to establish inter-tile connections.

Parameters:

  • index int Tile index; if outside the level, this is a no-op.
  • flags int Union of flags to assign.

Returns:

    uint Previous value of tile's working flags; 0 if no value has been assigned, or index is outside the level.

See also:

WipeFlags (col1, row1, col2, row2)
Clears all tile flags, in both the working and resolved sets, in a given region.

Neighboring tiles' flags remain unaffected. Flag resolution is not performed.

Parameters:

  • col1 int Column of one corner...
  • row1 int ...row of one corner...
  • col2 int ...column of another corner... (Columns will be sorted, and clamped.)
  • row2 int ...and row of another corner. (Rows too.)
